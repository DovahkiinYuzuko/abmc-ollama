# abmc_ollama_variable_function_spec_20260401_1530.md

## 1. i18n.py (多言語管理)
システム言語の自動取得、メッセージの翻訳管理、およびキャッシュ機能を担当する。

### クラス: `I18nManager`
* **`__init__(self)`**: 書き込み用キャッシュパスの設定、システムロケールの取得（デフォルトは 'en'）、およびメッセージ定義辞書の初期化を行う。
* **`_load_cache(self) -> Dict[str, Any]`**: `data/cache_languages.json` から翻訳キャッシュを読み込む。
* **`_save_cache(self)`**: 現在の翻訳キャッシュをファイルに保存する。
* **`get(self, key: str) -> str`**: 指定されたキーに対応するメッセージを返す。定義済みの辞書、キャッシュの順に探索し、見つからない場合は自動翻訳を実行する。
* **`_auto_translate(self, key: str) -> str`**: `GoogleTranslator` を使用して英語の原文から対象言語へ翻訳し、キャッシュに保存する。

### グローバル変数
* **`i18n`**: `I18nManager` のシングルトンインスタンス。

## 2. main.py (アプリケーション本体)
UIメニューの表示と各機能の実行制御を行う。

### クラス: `AbmcApp`
* **`__init__(self)`**: フィルタの初期化とロゴファイルのパス解決を行う。
* **`display_logo(self)`**: `logoAA.txt` を読み込み、アスキーアートを表示する。
* **`clear_screen(self)`**: OSに応じてターミナル画面を消去する。
* **`menu_run_chat(self)`**: モデル一覧を表示し、`ollama run` による対話を開始するメニュー。
* **`start_model(self, model_name)`**: 追加オプションを入力させ、モデルを実行する。
* **`menu_pull(self)`**: 新規モデルをダウンロードするメニュー。
* **`menu_remove(self)`**: 既存モデルを削除するメニュー。
* **`menu_status(self)`**: 現在実行中のモデル情報を表示するメニュー。
* **`menu_search_chat(self)`**: AI検索エージェントモードのモデル選択とセッション開始を行うメニュー。
* **`start_search_session(self, model_name)`**: 検索機能を伴うAIとの対話ループを実行する。
* **`menu_search_settings(self)`**: 検索エンジンの選択（Brave, Google等）とAPIキーの設定を行うメニュー。
* **`run(self)`**: アプリケーションのメインループ。

## 3. manager.py (バックエンド管理)
Ollamaコマンドの実行、検索APIの呼び出し、およびChainエージェントのロジックを担当する。

### クラス: `OllamaManager`
* **`__init__(self)`**: キャッシュや設定ファイルのパスを設定し、検索設定を読み込む。
* **`check_ollama(self)`**: `ollama` コマンドが利用可能か確認し、不可の場合は終了する。
* **`get_remote_models(self)`**: `ollama list` の実行結果（文字列）を取得する。
* **`load_cache(self)`**: モデルキャッシュを読み込む。
* **`save_cache(self, models_data, last_used=None)`**: 更新日時、最後に使用したモデル、モデルリストを保存する。
* **`load_search_config(self)`**: 検索エンジンの設定（APIキー等）を読み込む。
* **`save_search_config(self, config)`**: 検索設定をファイルに保存する。
* **`run_model(self, m, opt="")`**: 指定されたモデルを `ollama run` で起動する。
* **`pull_model(self, m)`**: モデルをダウンロードする。
* **`remove_model(self, m)`**: モデルを削除する。
* **`show_status(self)`**: `ollama ps` を実行し状態を表示する。
* **`web_search(self, query)`**: Brave API または DuckDuckGo を使用してウェブ検索を実行し、結果をリストで返す。
* **`_call_phase(self, model_name, system_prompt, user_content, is_json=True)`**: 特定のプロンプトで Ollama API を呼び出し、フェーズごとの処理結果を得る。
* **`chat_with_search(self, model_name, user_input, messages)`**: 下記の4ステップによるAI検索チャットを実行する。
    1.  **Planner**: 検索クエリを生成。
    2.  **Searcher**: ウェブ検索を実行。
    3.  **Extractor**: 検索結果から事実を抽出・検証。
    4.  **Writer**: 事実に基づき最終回答を執筆。

### グローバル変数
* **`ollama_mgr`**: `OllamaManager` のシングルトンインスタンス。

## 4. formatter.py (データ整形)
Ollamaの出力パースやフィルタリングを行う。

### クラス: `ModelFormatter`
* **`__init__(self)`**: サイズ単位（TB, GB, MB等）からバイト数への換算マップを定義する。
* **`parse_ollama_list(self, raw_text: str)`**: `ollama list` の生出力を解析し、モデル名・サイズ・バイト換算値を持つ辞書のリストを作成。サイズ順にソートして返す。
* **`_to_bytes(self, size_str: str) -> int`**: サイズ文字列を数値（バイト単位）に変換する。
* **`filter_models(self, models: list, keyword: str)`**: キーワードに合致する名前のモデルを抽出する。

### グローバル変数
* **`formatter`**: `ModelFormatter` のシングルトンインスタンス。

---

## 5. utils.py (汎用ユーティリティ)
パス解決やテキストのクレンジングを行う。

* **`get_resource_path(relative_path)`**: PyInstaller環境下での読み取り専用リソース（ロゴ等）のパスを取得する。
* **`get_writable_path(relative_path)`**: OSに応じた書き込み用ディレクトリ（AppData/Home）内のパスを取得し、ディレクトリを自動作成する。
* **`clean_text(text)`**: HTMLタグの除去、実体参照の復元、余計な空白の整理を行い、AIが読みやすい形式に整形する。

## 6. インストーラスクリプト (install_win.py / install_mac.py / install_linux.py)
プラットフォーム別のビルドとインストール設定を行う。

* **`run_command(command, description_jp, description_en)`**: コマンドを実行し、エラー時はシステムを終了する。
* **`install()`**: 下記手順を実行する。
    1.  旧ビルド産物（`build/`, `dist/`, `.spec`, 実行ファイル）の削除。
    2.  `requirements.txt` に基づく依存ライブラリのインストール。
    3.  `PyInstaller` による実行ファイルのビルド。
    4.  実行ファイルのプロジェクトルートへの移動と権限付与。
    5.  環境設定（Windows: `User PATH` への登録、Mac/Linux: `.zshrc`/`.bashrc` へのエイリアス追加）。
    6.  ビルド中間ファイルの削除。
