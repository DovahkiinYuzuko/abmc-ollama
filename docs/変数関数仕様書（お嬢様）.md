## 1. i18n.py (多言語管理)
システム言語の自動取得、メッセージの翻訳管理、およびキャッシュ機能を担当いたしますわ。

### クラス: I18nManager
* __init__(self): 書き込み用キャッシュパスの設定、システムロケールの取得（デフォルトは 'en'）、およびメッセージ定義辞書の初期化を行いますの。
* _load_cache(self) -> Dict[str, Any]: 「data/cache_languages.json」から翻訳キャッシュを読み込みますわ。
* _save_cache(self): 現在の翻訳キャッシュをファイルに保存いたします。
* get(self, key: str) -> str: 指定されたキーに対応するメッセージを返しますの。定義済みの辞書、キャッシュの順に探索し、見つからない場合は自動翻訳を実行いたしますわ。
* _auto_translate(self, key: str) -> str: 「GoogleTranslator」を使用して英語の原文から対象言語へ翻訳し、キャッシュに保存いたします。

### グローバル変数
* i18n: I18nManager のシングルトンインスタンスでございます。

## 2. main.py (アプリケーション本体)
UIメニューの表示と各機能の実行制御を行いますわ。

### クラス: AbmcApp
* __init__(self): フィルタの初期化とロゴファイルのパス解決を行いますの。
* display_logo(self): 「logoAA.txt」を読み込み、アスキーアートを表示いたしますわ。
* clear_screen(self): OSに応じてターミナル画面を消去いたします。
* menu_run_chat(self): モデル一覧を表示し、「ollama run」による対話を開始するメニューですわ。
* start_model(self, model_name): 追加オプションを入力させ、モデルを実行いたしますの。
* menu_pull(self): 新規モデルをダウンロードするメニューでございます。
* menu_remove(self): 既存モデルを削除するメニューですわ。
* menu_status(self): 現在実行中のモデル情報を表示するメニューでございます。
* menu_search_chat(self): AI検索エージェントモードのモデル選択とセッション開始を行うメニューですの。
* start_search_session(self, model_name): 検索機能を伴うAIとの対話ループを実行いたしますわ。
* menu_search_settings(self): 検索エンジンの選択（Brave, Google等）とAPIキーの設定を行うメニューですこと。
* run(self): アプリケーションのメインループにほかなりません。

## 3. manager.py (バックエンド管理)
Ollamaコマンドの実行、検索APIの呼び出し、およびChainエージェントのロジックを担当いたします。

### クラス: OllamaManager
* __init__(self): キャッシュや設定ファイルのパスを設定し、検索設定を読み込みますの。
* check_ollama(self): 「ollama」コマンドが利用可能か確認し、不可の場合は終了いたしますわ。
* get_remote_models(self): 「ollama list」の実行結果（文字列）を取得いたします。
* load_cache(self): モデルキャッシュを読み込みますの。
* save_cache(self, models_data, last_used=None): 更新日時、最後に使用したモデル、モデルリストを保存いたしますわ。
* load_search_config(self): 検索エンジンの設定（APIキー等）を読み込みますの。
* save_search_config(self, config): 検索設定をファイルに保存いたしますわ。
* run_model(self, m, opt=""): 指定されたモデルを 「ollama run」で起動いたします。
* pull_model(self, m): モデルをダウンロードいたしますの。
* remove_model(self, m): モデルを削除いたしますわ。
* show_status(self): 「ollama ps」を実行し状態を表示いたします。
* web_search(self, query): Brave API または DuckDuckGo を使用してウェブ検索を実行し、結果をリストで返しますの。
* _call_phase(self, model_name, system_prompt, user_content, is_json=True): 特定のプロンプトで Ollama API を呼び出し、フェーズごとの処理結果を得ますわ。
* chat_with_search(self, model_name, user_input, messages): 下記の4ステップによるAI検索チャットを実行いたします。
    1. Planner: 検索クエリを生成。
    2. Searcher: ウェブ検索を実行。
    3. Extractor: 検索結果から事実を抽出・検証。
    4. Writer: 事実に基づき最終回答を執筆。

### グローバル変数
* ollama_mgr: OllamaManager のシングルトンインスタンスでございます。

## 4. formatter.py (データ整形)
Ollamaの出力パースやフィルタリングを行いますわ。

### クラス: ModelFormatter
* __init__(self): サイズ単位（TB, GB, MB等）からバイト数への換算マップを定義いたしますの。
* parse_ollama_list(self, raw_text: str): 「ollama list」の生出力を解析し、モデル名・サイズ・バイト換算値を持つ辞書のリストを作成。サイズ順にソートして返しますわ。
* _to_bytes(self, size_str: str) -> int: サイズ文字列を数値（バイト単位）に変換いたします。
* filter_models(self, models: list, keyword: str): キーワードに合致する名前のモデルを抽出いたしますの。

### グローバル変数
* formatter: ModelFormatter のシングルトンインスタンスにございます。

---

## 5. utils.py (汎用ユーティリティ)
パス解決やテキストのクレンジングを行いますわ。

* get_resource_path(relative_path): PyInstaller環境下での読み取り専用リソース（ロゴ等）のパスを取得いたします。
* get_writable_path(relative_path): OSに応じた書き込み用ディレクトリ（AppData/Home）内のパスを取得し、ディレクトリを自動作成いたしますの。
* clean_text(text): HTMLタグの除去、実体参照の復元、余計な空白の整理を行い、AIが読みやすい形式に整形いたしますわ。

## 6. インストーラスクリプト (install_win.py / install_mac.py / install_linux.py)
プラットフォーム別のビルドとインストール設定を行いますの。

* run_command(command, description_jp, description_en): コマンドを実行し、エラー時はシステムを終了いたしますわ。
* install(): 下記手順を実行いたします。
    1. 旧ビルド産物（「build/」、「dist/」、「.spec」、実行ファイル）の削除。
    2. 「requirements.txt」に基づく依存ライブラリのインストール。
    3. 「PyInstaller」による実行ファイルのビルド。
    4. 実行ファイルのプロジェクトルートへの移動と権限付与。
    5. 環境設定（Windows: 「User PATH」への登録、Mac/Linux: 「.zshrc」/「.bashrc」へのエイリアス追加）。
    6. ビルド中間ファイルの削除。
